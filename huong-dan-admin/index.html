2. Trước khi bắt đầu — Kiểm tra sức khỏe

Bước 1: Đăng nhập admin

  1. Mở trình duyệt (Chrome hoặc Safari)
  2. Vào https://www.nguyenlananh.com/admin/login/
  3. Nhập email + mật khẩu admin
  4. Nhấn "Đăng nhập"
  5. Nếu có widget "Tôi không phải robot" — hãy tick vào
⚠️ Lưu ý: Nếu sai mật khẩu 5 lần, tài khoản sẽ bị khóa 15 phút.

Bước 3: Đổi mật khẩu (QUAN TRỌNG)

  1. Vào Dashboard admin
  2. Tìm nút "Đổi mật khẩu"
  3. Nhập mật khẩu hiện tại + mật khẩu mới
  4. Mật khẩu mới nên: tối thiểu 12 ký tự, có chữ hoa, chữ thường, số, ký tự đặc biệt
  5. Lưu mật khẩu trong password manager (1Password, Bitwarden, hoặc ghi giấy an toàn)

3. Quản trị Admin

3.1. Tạo tài khoản admin mới

Khi nào cần: Khi muốn thêm người quản trị khác.

  1. Vào /admin/
  2. Tìm form "Tạo admin user"
  3. Nhập email, tên hiển thị, vai trò
  4. Vai trò: super_admin (toàn quyền), ops_manager (vận hành), content_manager (nội dung), support_agent (hỗ trợ)
  5. Nhấn "Tạo" → hệ thống gửi mật khẩu tạm qua email

4. Quản lý thành viên

4.1. Xem danh sách thành viên

Vào /admin/members/ — bạn sẽ thấy bảng gồm: Email, Tên, Loại membership, Ngày đăng ký, Trạng thái.

4.2. Cấp membership thủ công

  1. Vào /admin/members/
  2. Tìm user → nhấn "Edit"
  3. Đổi membership_type thành: year1, year2, year3, hoặc lifetime
  4. Nhấn "Save"

4.3. Cấp quyền truy cập sản phẩm

  1. Vào /admin/members/ → tìm user → nhấn "Edit"
  2. Trong phần content_access, thêm product code (xem đầy đủ trong /admin/audit/ → Product Inventory)
  3. Nhấn "Save"
⚠️ CẢNH BÁO: Xóa user = mất toàn bộ dữ liệu. Không thể hoàn tác. Khuyến nghị: set active = 0 thay vì xóa.

5. Quản lý đơn hàng & thanh toán

5.1. Xem đơn hàng

Vào /admin/payments/ — bạn sẽ thấy: Mã đơn, Email khách, Sản phẩm, Số tiền, Provider, Trạng thái.

5.2. Trạng thái đơn hàng

Trạng tháiÝ nghĩaHành động?
pendingĐang chờ thanh toánKhông — khách đang thanh toán
paid / captured / completedĐã thanh toánKhông — hệ thống tự cấp quyền
failed / denied / canceledThất bạiLiên hệ khách nếu cần
refunded / reversedĐã hoàn tiềnHệ thống tự thu hồi quyền

5.3. Hoàn tiền (Refund)

  1. Vào /admin/payments/ → tìm đơn → nhấn "Refund" → xác nhận
  2. Hệ thống tự: hoàn tiền qua PayPal, thu hồi quyền truy cập, gửi email cho khách

6. Quản lý nội dung

6.1. Bài viết

Bài viết là file HTML tĩnh trong /bai-viet/. Mỗi bài là 1 thư mục con. Để thêm bài mới: cần người kỹ thuật giúp, hoặc dùng Creator Dashboard.

6.2. Sản phẩm & giá

Xem danh sách tại /admin/audit/ → "Product Inventory". Đổi giá: cần người kỹ thuật sửa trong constants.js, sau đó chạy npm run validate:prices.

6.3. Kiểm tra broken links

Chạy: npm run validate:links — nếu PASS → không có link hỏng.

7. Quản lý Creator

7.1. Duyệt đơn ứng tuyển

  1. Vào /admin/creator-applications/
  2. Đọc đơn → chọn: Approve (duyệt) / Reject (từ chối) / Request changes (yêu cầu sửa)
  3. Hệ thống tự gửi email thông báo cho Creator

7.2. Chính sách Creator

Đọc tại /creators/policy/: IP (Creator giữ 100% bản quyền), Revenue share 70/30, Thanh toán hàng tháng (tối thiểu 500.000 VND hoặc $20).

8. Báo cáo Audit & Thống kê

Chỉ số chính trong /admin/audit/

  • MRR — Doanh thu định kỳ hàng tháng
  • ARR — Doanh thu định kỳ/năm
  • Revenue 30 ngày — Tổng doanh thu 30 ngày qua
  • Conversion funnel — Tỷ lệ: user → paid → học xong → nộp bài
  • Recent webhook events — 20 sự kiện thanh toán mới nhất
  • Revenue by month — Doanh thu theo tháng
  • New users 30d/7d — Người dùng mới

9. Bảo mật — 2FA & Turnstile

9.1. Bật 2FA

  1. Đăng nhập thành viên tại /members/
  2. Vào /members/security/
  3. Nhấn "Bật 2FA" → tải Google Authenticator → quét QR → nhập mã 6 số
  4. Lưu mã dự phòng ở nơi an toàn

9.2. Mất điện thoại?

  1. Dùng mã dự phòng đã lưu
  2. Vào /members/verify-2fa/ → mở "Dùng mã dự phòng" → nhập 1 mã
⚠️ Nếu mất cả điện thoại LẪN mã dự phòng → liên hệ kỹ thuật để reset thủ công.

9.3. Turnstile (Bot Protection)

⚠️ YÊU CẦU HÀNH ĐỘNG: Turnstile hiện đang dùng test key (chưa bảo vệ thật). Bạn cần tạo widget thật trên Cloudflare dashboard:
1. Vào dash.cloudflare.com → Turnstile → Add Widget
2. Widget name: nguyenlananh-prod
3. Domain: nguyenlananh.com
4. Mode: Managed
5. Tạo xong → copy Site Key (bắt đầu 0x4...) và Secret Key
6. Set Secret Key: npx wrangler pages secret put TURNSTILE_SECRET_KEY --project-name=nguyenlananh-com
7. Sửa Site Key trong /assets/turnstile-config.js → thay 1x00000000000000000000AA bằng site key thật
8. Deploy lại → test: token "dummy" phải trả TURNSTILE_FAILED
Cho đến khi làm xong, tất cả form có Turnstile sẽ BỊ KHÓA (fail closed).

11. Xử lý sự cố thường gặp

Sự cố 1: "Không đăng nhập được admin"

  • Sai email/mật khẩu → Kiểm tra lại
  • Khóa 5 lần sai → Chờ 15 phút
  • Tài khoản bị khóa → Liên hệ super_admin khác
  • Widget Turnstile không hiện → Tắt ad blocker

Sự cố 2: "Khách báo không nhận email"

  • Kiểm tra thư rác/spam
  • Kiểm tra email đúng trong /admin/members/
  • Kiểm tra /admin/audit/ → site_events

Sự cố 3: "Đã thanh toán nhưng không có quyền truy cập"

  • Vào /admin/payments/ → tìm đơn hàng
  • Nếu pending → chờ webhook (vài phút)
  • Nếu paid mà không có quyền → cấp thủ công (mục 4.3)
  • Nếu failed → khách cần thanh toán lại

Sự cố 4: "Website chậm hoặc không mở"

Kiểm tra https://www.cloudflarestatus.com/ — nếu Cloudflare OK → liên hệ kỹ thuật.

Sự cố 5: "Lỗi TURNSTILE_FAILED"

Khách thử: refresh trang, tắt VPN, tắt ad blocker. Nếu vẫn lỗi → liên hệ kỹ thuật.

12. Liên hệ kỹ thuật

Khi nào cần liên hệ?

  • Website không hoạt động
  • Lỗi không tự khắc phục được
  • Cần thêm tính năng mới
  • Cần đổi giá sản phẩm
  • Cần tạo bài viết mới (HTML)
  • Mất 2FA + mất mã dự phòng

Thông tin cần cung cấp khi báo lỗi

  1. Bạn đang làm gì? (ví dụ: "Tôi đang duyệt đơn Creator")
  2. Trang nào? (ví dụ: /admin/creator-applications/)
  3. Lỗi hiển thị gì? (chụp màn hình nếu được)
  4. Thời gian xảy ra? (ngày/giờ)
  5. Email user liên quan (nếu có)
Bảo mật — KHÔNG BAO GIỜ:
❌ Chia sẻ mật khẩu admin cho ai
❌ Gửi credentials qua chat/email
❌ Cho ai khác quyền super_admin nếu không cần thiết
❌ Sửa code nếu không biết dev
NÊN:
✅ Đổi mật khẩu định kỳ 3 tháng/lần
✅ Bật 2FA cho tài khoản admin
✅ Kiểm tra audit log hàng tuần
