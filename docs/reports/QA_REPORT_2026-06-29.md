### 4. Pro Layer Entitlement (P0) — FIXED
**Before:** Pro packs required `isMembershipActive(user)` → only year1/2/3 paid members could access. No individual purchase flow existed.
**After:**
- Each Pro pack requires specific `content_access` (`pro_reset`, etc.) OR active membership
- Middleware checks content_access table for individual pack purchases
- Anonymous users → `/join/?next_path=...`
- Logged-in non-entitled → Pro index with `?checkout=<slug>&plan=<code>` → opens inline checkout modal
- **Files:** `functions/_middleware.js`, `assets/pro-layer.js`

### 5. Checkout Email (P0) — FIXED
**Before:** Always required `body.email` — no session awareness.
**After:**
- Backend: `createCheckoutResponse` parses session cookie → uses `session.email` if available
- Frontend: `products-checkout.js` fetches `/api/auth/session` → hides email input for logged-in users, shows session email
- Guest checkout still works (email input shown)
- **Files:** `functions/_lib/payments.js`, `assets/products-checkout.js`

### 6. Pro Page UI (P0) — FIXED
**Before:** "Mở gói" links → gated page → redirect loop back to index.
**After:**
- Dynamic button states via `pro-layer.js`: "Đăng nhập để mở gói" / "Mua gói" / "Vào gói"
- Inline checkout modal with VietQR/PayPal selector, Turnstile, QR display
- Both VI and EN versions updated
- **Files:** `members/pro/index.html`, `en/members/pro/index.html`, `assets/pro-layer.js`

### 7. Fulfillment (P0) — FIXED
**Before:** `pro_*` plans treated as membership upgrades (changed `membership_type`).
**After:** `pro_` prefix added to `isPremium` check → Pro packs grant `content_access` without changing user's membership tier.
**File:** `functions/_lib/payments.js`
